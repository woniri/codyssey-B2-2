# 워크플로우 설명서

## 프로젝트명

RSS 기반 AI 기술 뉴스 자동 수집 및 요약 시스템

---

# 1. 워크플로우 개요

본 프로젝트는 RSS를 이용하여 최신 AI 기술 뉴스를 자동으로 수집하고, 생성형 AI를 활용하여 기사 내용을 요약 및 감성 분석한 뒤 AI 기반 썸네일 이미지를 생성하여 Notion 데이터베이스에 자동 저장하는 자동화 시스템이다.

또한 URL 기반 중복 검사, 하루 1건 저장 정책, Webhook 기반 API 재시도 기능을 적용하여 실제 운영 환경을 고려한 안정적인 자동화 파이프라인을 구현하였다.

# 5. 메인 시나리오 구성

| No | Make 모듈 | 주요 기능 |
|---|---|---|
|①|Scheduler|매일 지정된 시간 자동 실행|
|②|Google Sheets|RSS 목록 조회|
|③|RSS → Retrieve RSS Feed Items|최신 기사 **10건** 조회|
|④|🟢 Filter (제목 검증)|`exists()`를 이용하여 제목 존재 여부 확인|
|⑤|🟢 Filter (AI 기사 선별)|AI 관련 기사만 통과|
|⑥|Notion → Search Objects|원문 URL 중복 검사|
|⑦|🟢 Filter (중복 제거)|중복이 없는 기사만 통과|
|⑧|Notion → Search Objects|저장일자 조회|
|⑨|🟢 Filter (하루 1건 저장)|당일 저장 이력이 없는 경우만 통과|
|⑩|HTTP|Gemini 하위 시나리오 호출|
|⑪|JSON → Parse JSON|Gemini 결과 파싱|
|⑫|Stability AI|썸네일 생성|
|⑬|Google Drive|이미지 업로드|
|⑭|Google Drive|공유 링크 생성|
|⑮|Notion|뉴스 저장|

# 7. 주요 모듈 설명

---

## ① Scheduler

|항목|설정|
|---|---|
|모듈|Scheduler|
|실행 주기|**매일 지정된 시간**|

### 역할

자동으로 워크플로우를 시작한다.

> 💡 **설계 포인트**
>
> 사용자의 개입 없이 매일 동일한 시간에 최신 뉴스를 자동 수집하기 위해 Scheduler를 사용하였다.

---

## ② Google Sheets

|항목|설정|
|---|---|
|모듈|Search Rows|
|관리 데이터|RSS URL 목록|

### 역할

RSS 주소를 중앙에서 관리한다.

> 💡 **설계 포인트**
>
> RSS 주소를 Google Sheets에서 관리하여 새로운 뉴스 사이트를 추가하거나 삭제할 때 Make 시나리오를 수정하지 않아도 되도록 설계하였다.

---

## ③ RSS Feed

|항목|설정|
|---|---|
|모듈|Retrieve RSS Feed Items|
|Max Items|**10**|

### 출력

- 기사 제목
- 기사 링크
- 기사 요약
- 발행일시

> 💡 **설계 포인트**
>
> 최신 기사 1건만 가져오면 AI 관련 기사가 없을 수 있으므로 **최신 10건을 조회한 뒤 AI 기사만 선별**하도록 구성하였다.

---

## ④ 🟢 Filter (제목 검증)

|항목|설정|
|---|---|
|조건|`exists()`|
|검사 대상|기사 제목|

### 역할

제목이 존재하는 기사만 다음 단계로 전달한다.

> 💡 **설계 포인트**
>
> RSS 피드에 따라 제목이 비어 있는 데이터가 존재할 수 있으므로 `exists()` 조건을 사용하여 유효한 기사만 처리하도록 구성하였다.

---

## ⑤ 🟢 Filter (AI 기사 선별)

|항목|설정|
|---|---|
|검사 대상|제목 + 요약문|
|키워드|**AI, LLM, OpenAI, 인공지능, 생성형 AI**|

### 역할

AI 관련 기사만 선별한다.

> 💡 **설계 포인트**
>
> AI 관련 기사만 처리하여 불필요한 Gemini API 호출을 줄이고 프로젝트 목적에 맞는 기사만 저장하도록 설계하였다.

---

## ⑥ Notion Search (URL)

|항목|설정|
|---|---|
|검색 기준|원문 URL|

### 역할

중복 여부를 확인한다.

> 💡 **설계 포인트**
>
> 동일 기사의 반복 저장을 방지하고 API 호출 비용을 줄이기 위해 URL을 기준으로 중복 여부를 검사하였다.

---

## ⑦ 🟢 Filter (중복 제거)

검색 결과가 없는 기사만 다음 단계로 전달한다.

> 💡 **설계 포인트**
>
> 중복 기사는 즉시 제외하여 불필요한 후속 작업을 수행하지 않도록 구성하였다.

---

## ⑧ Notion Search (저장일자)

|항목|설정|
|---|---|
|검색 기준|저장일자|

### 역할

오늘 저장된 뉴스가 있는지 확인한다.

---

## ⑨ 🟢 Filter (하루 1건 저장)

오늘 저장된 데이터가 없는 경우만 저장한다.

> 💡 **설계 포인트**
>
> 과제 요구사항인 하루 1건 저장 정책을 구현하기 위해 저장일자를 기준으로 필터링하였다.

---

## ⑩ HTTP (Webhook)

|항목|설정|
|---|---|
|Method|**POST**|
|Body|**JSON**|

### 역할

하위 시나리오의 Gemini를 호출한다.

> 💡 **설계 포인트**
>
> Gemini를 하위 시나리오로 분리하여 유지보수성을 높이고 API 오류 발생 시 최대 2회 재시도할 수 있도록 설계하였다.

---

## ⑪ Google Gemini

|항목|설정|
|---|---|
|출력 형식|**JSON**|

### 생성 데이터

- 뉴스 요약
- 감성 분석
- 이미지 프롬프트

> 💡 **설계 포인트**
>
> JSON 형식으로 결과를 반환하여 Parse JSON 모듈에서 데이터를 쉽게 활용할 수 있도록 구성하였다.

---

## ⑫ Parse JSON

Gemini 응답을

- summary
- sentiment
- image_prompt

로 분리한다.

---

## ⑬ Stability AI

|항목|설정|
|---|---|
|입력|image_prompt|
|출력|PNG|

> 💡 **설계 포인트**
>
> 기사 내용을 직관적으로 표현할 수 있는 썸네일 이미지를 자동 생성하였다.

---

## ⑭ Google Drive

|항목|설정|
|---|---|
|공유 권한|**Anyone with Link**|

> 💡 **설계 포인트**
>
> Notion에서 이미지를 사용할 수 있도록 공유 가능한 링크를 생성하였다.

---

## ⑮ Notion Database

### 저장 항목

- 제목
- 요약문
- 감성 태그
- 원문 링크
- 발행일시
- 저장일자
- 이미지 링크

> 💡 **설계 포인트**
>
> URL 기반 중복 검사와 저장일자 검사를 함께 적용하여 **중복 저장 방지**와 **하루 1건 저장 정책**을 동시에 만족하도록 구현하였다.

# 8. 예외 처리 구조

| 상황 | 처리 방식 |
|------|-----------|
| RSS 기사 없음 | 실행 종료 |
| 제목 없음 | 기사 제외 |
| URL 중복 | 저장하지 않음 |
| 당일 저장 이력 존재 | 저장하지 않음 |
| Gemini API 실패 | **60초 간격 최대 2회 재시도** |
| 이미지 생성 실패 | 이미지 없이 기사 저장 계속 진행 |

# 9. 프로젝트 구현 성과

본 프로젝트를 통해 RSS 기반 뉴스 수집부터 생성형 AI를 활용한 기사 요약, 감성 분석, 이미지 생성, Google Drive 저장, Notion 자동 저장까지 전 과정을 하나의 자동화 워크플로우로 구현하였다.

또한 다음과 같은 기능을 성공적으로 구현하였다.

- RSS 기반 최신 AI 기술 뉴스 자동 수집
- AI 관련 기사 자동 필터링
- Google Gemini 기반 3줄 요약
- 감성 분석 자동 수행
- Stability AI 기반 썸네일 생성
- Google Drive 자동 업로드 및 공유 링크 생성
- Notion 데이터베이스 자동 저장
- URL 기반 중복 기사 제거
- 하루 1건 저장 정책 적용
- Webhook 기반 최대 2회 API 재시도
- 사람의 개입 없이 전 과정 자동 실행
