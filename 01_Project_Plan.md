## 프로젝트 기획서

---

# 1. 프로젝트 개요

## 프로젝트명

**RSS 기반 AI 기술 뉴스 자동 수집 및 요약 시스템**

## 프로젝트 목적

최근 AI 기술의 발전 속도가 매우 빨라지면서 매일 수많은 관련 뉴스가 생성되고 있다. 그러나 다양한 뉴스 사이트와 커뮤니티를 직접 방문하여 정보를 수집하고 정리하는 작업은 많은 시간과 노력이 필요하다.

본 프로젝트는 RSS 피드를 활용하여 최신 기술 뉴스를 자동으로 수집하고, 생성형 AI를 이용하여 핵심 내용을 요약한 뒤 협업 도구인 Notion에 자동 저장하는 자동화 시스템을 구축하는 것을 목표로 한다.

또한 단순한 뉴스 수집이 아닌 AI 관련 기사만 선별하고, 감성 분석과 AI 기반 이미지 생성 기능까지 추가하여 활용도를 높였다.

전체 과정은 사람이 개입하지 않아도 자동으로 동작하도록 설계하였다.

# 5. 워크플로우 상세 설명

## 5-1. RSS 뉴스 수집

Google Sheets에 등록된 RSS 주소 목록을 읽어 최신 기술 뉴스를 자동으로 수집한다.

RSS Feed 모듈을 통해 기사 제목, 링크, 요약문, 발행일시를 가져오며 이후 단계의 입력 데이터로 활용한다.

---

## 5-2. 주제 필터링

### 선정 배경

기술 뉴스는 AI, 반도체, 클라우드, 보안, 모바일, 게임, IoT 등 다양한 분야의 기사가 지속적으로 생성된다.

모든 기사를 대상으로 생성형 AI를 호출할 경우 불필요한 API 호출이 증가하고 처리 시간이 길어질 뿐만 아니라 프로젝트 목적과 관련 없는 기사까지 저장되는 문제가 발생한다.

본 프로젝트에서는 **생성형 AI 기술 동향을 효율적으로 수집하고 분석하는 것**을 핵심 목표로 설정하였다.

최근 생성형 AI는 산업 전반에서 가장 빠르게 발전하고 있는 분야이며, 새로운 모델과 서비스가 지속적으로 발표되고 있다. 또한 개발자뿐만 아니라 일반 사용자에게도 활용 가치가 매우 높은 분야이므로 프로젝트 주제로 적합하다고 판단하였다.

따라서 AI 관련 기사만 자동 선별하여 저장하도록 설계하였다.

---

### 필터링 기준

RSS로 수집된 기사의 **제목과 요약문**을 대상으로 키워드 기반 필터링을 수행하였다.

다음 키워드 중 하나 이상이 포함될 경우 AI 관련 기사로 판단한다.

| 키워드        | 선정 이유                                   |
| ---------- | --------------------------------------- |
| **AI**     | 인공지능 관련 기사에서 가장 많이 사용되는 핵심 키워드          |
| **LLM**    | 대규모 언어모델(Large Language Model) 관련 기사 선별 |
| **OpenAI** | ChatGPT 등 대표적인 생성형 AI 서비스 관련 기사 수집      |
| **인공지능**   | 국내 기술 기사에서 가장 일반적으로 사용되는 공식 용어          |
| **생성형 AI** | 생성형 AI 모델 및 서비스 관련 기사 선별                |

---

### 적용 효과

* AI 관련 기사만 자동 선별
* 일반 IT 기사 제거
* 불필요한 AI API 호출 감소
* 요약 품질 향상
* 처리 시간 단축
* 프로젝트 목적에 맞는 데이터만 저장

---

## 5-3. 중복 기사 방지

동일한 뉴스가 반복 저장되지 않도록 원문 링크(URL)를 기준으로 중복 여부를 확인한다.

이미 저장된 기사인 경우 이후 작업을 수행하지 않고 종료한다.

---

## 5-4. 하루 1건 저장

과제 요구사항에 따라 하루에 1건의 뉴스만 저장되도록 구현하였다.

노션 데이터베이스에 저장일자를 함께 기록하고, 저장 전 저장일자를 조회하여 동일 날짜에 저장된 데이터가 존재할 경우 이후 기사는 저장하지 않는다.

---

## 5-5. AI 요약 및 감성 분석

Google Gemini를 이용하여 뉴스 내용을 분석한다.

생성 결과

* 뉴스 3줄 요약
* 감성 분석
* 이미지 생성 프롬프트

감성 분류

* 긍정
* 부정
* 중립

---

## 5-6. AI 이미지 생성

Stability AI를 활용하여 기사 내용을 기반으로 한 썸네일 이미지를 생성한다.

생성된 이미지는 Google Drive에 저장하고 공유 링크를 생성하여 Notion에 함께 저장한다.

---

## 5-7. Notion 자동 저장

최종 결과를 Notion 데이터베이스에 자동 저장한다.

저장 항목

* 제목
* 요약문
* 감성 태그
* 원문 링크
* 발행일시
* 저장일자
* 썸네일 이미지 링크

---

# 6. 에러 처리 정책

## 6-1. RSS 수집 실패

RSS에서 기사를 가져오지 못한 경우 해당 실행을 종료한다.

**선택 이유**

후속 단계에서 빈 데이터를 처리하는 오류를 방지하기 위함이다.

---

## 6-2. Gemini API 실패

Gemini API 호출 실패 시 Webhook 기반 재시도 구조를 이용하여 최대 2회까지 자동 재시도한다.

```text
Gemini 실패
      │
      ▼
60초 대기
      │
      ▼
1차 재시도
      │
      ▼
실패
      │
      ▼
60초 대기
      │
      ▼
2차 재시도
```

모든 재시도에 실패할 경우 Discord로 오류 알림을 전송한다.

**선택 이유**

일시적인 API 장애나 응답 지연 상황에 대응하여 시스템 안정성을 높이기 위함이다.

---

## 6-3. 이미지 생성 실패

이미지 생성에 실패하더라도 뉴스 저장은 정상적으로 진행되도록 별도의 예외 처리 경로를 구성하였다.

**선택 이유**

이미지 생성 오류로 인해 전체 워크플로우가 중단되는 것을 방지하기 위함이다.

---

## 6-4. 중복 기사 처리

이미 저장된 기사인 경우 즉시 종료하여 불필요한 AI 호출을 방지한다.

**선택 이유**

API 호출 비용 절감 및 데이터 중복 방지를 위함이다.

---

## 6-5. Discord 오류 알림

워크플로우 주요 단계에서 오류가 발생할 경우 Discord 채널로 자동 알림을 전송한다.

오류 알림 대상

* RSS 수집 실패
* Gemini API 호출 실패
* 이미지 생성 실패
* Notion 저장 실패

전송 내용

* 오류 발생 모듈
* 오류 메시지
* 발생 시간
* 상세 오류 내용

**선택 이유**

자동화 시스템 운영 중 발생하는 문제를 빠르게 확인하고 대응하기 위함이다.

---

# 7. 팀 역할 및 개인별 작업 요약

## 협업 방식

본 프로젝트는 6인 팀으로 진행하였다.

Make 자동화 시나리오는 여러 사람이 동시에 수정할 경우 버전 충돌과 테스트 결과 불일치가 발생할 가능성이 높다고 판단하였다.

따라서 모든 팀원이 동일한 프로젝트를 기준으로 개별 자료 조사와 설계, 테스트를 수행한 후 팀 회의를 통해 각자의 결과를 비교·분석하였다.

최종적으로 가장 적합한 설계안을 선정하였으며, 선정된 설계안을 기반으로 **팀원 1(이원일)**이 Make 시나리오를 최종 구현하고 전체 시스템을 통합하였다.

# 8. 최종 결과

본 프로젝트는 RSS를 활용한 기술 뉴스 자동 수집부터 생성형 AI 기반 요약, 감성 분석, 이미지 생성, Google Drive 저장, Notion 자동 저장까지 전 과정을 자동화하였다.

또한 AI 관련 기사 자동 필터링, 중복 기사 방지, 하루 1건 저장 제한, Webhook 기반 API 재시도, Discord 오류 알림 기능을 적용하여 안정적으로 운영 가능한 자동화 시스템을 구축하였다.

이를 통해 데이터 수집 → AI 가공 → 저장으로 이어지는 전체 자동화 파이프라인을 성공적으로 설계하고 구현하였다.
