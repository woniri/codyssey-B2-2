## 의사결정 8. 이미지 생성 실패 처리

### 검토 안건

* 이미지 생성 실패 시 전체 워크플로우 종료
* 이미지 생성 실패 시 기사 저장은 계속 진행

### 최종 결정

**이미지 생성에 실패하더라도 기사 저장은 계속 진행한다.**

### 선정 이유

이미지 생성은 뉴스 정보를 시각적으로 보완하는 부가 기능이며, 기사 요약 및 데이터 저장은 프로젝트의 핵심 기능이다.

외부 이미지 생성 API는 일시적인 오류가 발생할 수 있으므로, 이미지 생성 실패를 전체 워크플로우 실패로 처리하지 않고 핵심 데이터는 계속 저장하도록 결정하였다.

### 구현 결과

* 이미지 생성 실패 시에도 기사 제목, 요약, 감성 분석, 원문 링크, 발행일시 등 핵심 데이터는 정상적으로 Notion에 저장되도록 구성하였다.
* 이미지 데이터가 없는 경우에도 저장 과정에서 오류가 발생하지 않도록 예외 상황을 고려하여 안정적으로 처리하였다.
